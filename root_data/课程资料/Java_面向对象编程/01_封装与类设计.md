## 二、核心知识点

### 2.1 什么是封装

封装（Encapsulation）将**数据**与**操作数据的方法**绑定在类中，并限制外部对内部状态的直接访问。

| 好处 | 说明 |
|------|------|
| 数据安全 | 避免外部随意修改对象状态 |
| 易维护 | 内部实现可改，对外接口不变 |
| 易用 | 调用者只需关心公开 API |

### 2.2 访问修饰符

| 修饰符 | 同类 | 同包 | 子类 | 任意 |
|--------|------|------|------|------|
| `private` | ✓ | ✗ | ✗ | ✗ |
| `default` | ✓ | ✓ | ✗ | ✗ |
| `protected` | ✓ | ✓ | ✓ | ✗ |
| `public` | ✓ | ✓ | ✓ | ✓ |

**实践建议**：字段默认 `private`；对外只暴露必要 `public` 方法。

### 2.3 构造方法与 this

- 构造方法名与类名相同，无返回值类型。  
- 可重载多个构造方法，通过 `this(...)` 调用其他构造器。  
- 未写构造器时，编译器提供无参默认构造器；一旦自定义构造器，默认构造器不再自动生成。

### 2.4 静态成员

- `static` 字段/方法属于**类**，不属于某个对象。  
- 典型用途：常量 `public static final`、工具方法、计数器。  
- 静态方法中不能直接访问非静态成员（无 `this`）。

## 三、代码示例：银行账户类

```java
public class BankAccount {
    private final String accountNo;
    private double balance;
    private static int counter = 0;

    public BankAccount(String owner) {
        this.accountNo = "ACC" + (++counter);
        this.balance = 0.0;
    }

    public void deposit(double amount) {
        if (amount <= 0) {
            throw new IllegalArgumentException("存款金额必须为正");
        }
        balance += amount;
    }

    public boolean withdraw(double amount) {
        if (amount <= 0 || amount > balance) {
            return false;
        }
        balance -= amount;
        return true;
    }

    public double getBalance() {
        return balance;
    }

    public String getAccountNo() {
        return accountNo;
    }
}
```

**设计要点**：

- `balance` 私有，外部不能 `account.balance = -100`。  
- `withdraw` 返回 `boolean` 表示是否成功，比抛异常更轻量（也可按项目规范选择异常）。  
- `accountNo` 用 `final` 保证创建后不可变。

## 四、练习题

### 4.1 概念题

1. 封装是如何实现的？至少写出两点。  
2. `protected` 与 `default` 的区别是什么？  
3. 为什么要把成员变量设为 `private` 并提供 getter/setter？

### 4.2 编程题

**题 1：Person 类**  
定义 `Person`：字段 `name`（String）、`age`（int），要求：

- 构造方法传入 name、age，age 小于 0 时设为 0；  
- 提供 getter；`setAge` 仅允许 0～150；  
- `introduce()` 返回 `"我叫xxx，今年xx岁"`。

**题 2：Book 与 Library**  
- `Book`：title、author、isBorrowed；方法 `borrow()`、`returnBack()`。  
- `Library`：维护 `Book[]` 或 `ArrayList<Book>`，提供 `addBook`、`findByTitle`、`listAvailable`。

**题 3：日期类 Lite**  
实现 `SimpleDate`（year, month, day），校验合法性，提供 `isLeapYear()`、`daysInMonth()`、`toString()` 格式 `yyyy-MM-dd`。

### 4.3 参考答案要点（题 1 片段）

```java
public class Person {
    private String name;
    private int age;

    public Person(String name, int age) {
        this.name = name;
        this.age = Math.max(0, age);
    }

    public void setAge(int age) {
        if (age >= 0 && age <= 150) {
            this.age = age;
        }
    }

    public String introduce() {
        return "我叫" + name + "，今年" + age + "岁";
    }
}
```

## 五、常见错误

| 错误 | 说明 |
|------|------|
| 公开可变字段 | `public double balance` 破坏封装 |
| 在 setter 中不做校验 | 导致非法状态进入对象 |
| 滥用 static | 将与对象状态相关的方法标为 static |
