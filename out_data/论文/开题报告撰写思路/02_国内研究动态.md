# 2. 国内外研究现状（国内部分）

围绕大语言模型与生成式人工智能在程序设计教学中的应用，国内学者已从课程教学模式、教学实践评估与智能系统构建等层面开展了较多探索。

在课程教学模式方面，徐建军等在计算机程序设计课程中持续探索大语言模型辅助教学的具体做法[5]。李姗姗等提出以“学—用—思—评”闭环设计为核心的综合教学模式，并在 C++ 编程教学中开展理论认知、实践应用与思维提升三阶段教学实践[2]。梁家健等研究融合大语言模型的三元一站式编程教学模式[16]。李兴隆等将编程协同方式从“人人结对”拓展为“人机结对”，对大模型赋能编程教学进行实践与反思[23]。上述研究丰富了程序设计课程的组织实施方式，但多聚焦于单门课程与课堂流程，在可部署的 Web 助教平台及跨课程复用方面仍相对薄弱。

在实践落地层面，丁文婷等开展基于 LLM 的人机协同项目式编程教学创新实践[6]；唐琳等、王栋等分别探讨大语言模型在计算机编程实践课程中的教学策略与应用路径[8][17]。杨江英等从问题、策略与案例角度梳理生成式人工智能支持下的编程教育实施要点[22]。兰丽娜等构建融合生成式 AI 的智能编程训练系统 AIOJ，并通过实验组与对照组对比，说明其在学生编程能力、学习投入度及批判性思维等方面具有积极影响[13]。然而，相关实践多围绕单一教学场景或训练评测展开，对课程知识库约束、回答引用溯源及管理员侧 AI 资源治理等工程化能力关注仍显不足。

在系统平台建设层面，郭菲等构建了包含智能问答、代码诊断与学习路径规划等功能的个性化助教平台，并在教学过程中开展实证研究[7]。孙彦武等基于课程知识库与检索增强生成技术，构建 Python 程序设计智能问答系统[1]；潘耀宗等结合 RAG 技术设计计算机实验指导平台，以 Prompt 模板与知识检索提升实验效率[27]。陈玲等探索基于大模型智能体的代码分析与评测系统研究与应用[4]。上述成果表明国内智能辅导系统建设已取得一定进展，但多聚焦单点功能，尚未普遍实现课程 RAG、结构化代码讲解、学情分析与教师/管理员治理的一体化闭环。因此，有必要在吸收上述研究成果的基础上，设计面向高校程序设计教学的集成式智能助教系统（本课题“慧编学伴”）。
