# 1. 研究背景

以大语言模型为代表的人工智能技术给计算机教育带来了前所未有的机遇和挑战，“AI for 教学”已成为当前课程改革的重要方向，算法类、数据类、系统类课程均已开始探索具体实践路径；其中，程序设计类课程作为计算机专业最基础、实践性最强、与就业结合最紧密的课程，更应当作为改革的重点方向[25]。与此同时，高等教育正加速推进教育数字化转型，面向“人工智能+X”复合型人才培养需求，基于大语言模型的智能助教平台逐步进入教学实践，为人工智能技术赋能教育数字化转型提供了新的范式[7]。在此背景下，程序设计类课程如何在新技术条件下实现教学组织与辅导方式的系统升级，成为亟待回答的问题。

然而，现有程序设计课程仍普遍面临结构性矛盾。相关研究指出，传统编程教学以教师为中心，难以兼顾学生个性化需求，评价体系相对单一，教学内容更新滞后，已难以充分适应技术快速迭代的要求[2]。以 C 语言程序设计等基础课程为例，个性化支持不足、课程教学与实际编码场景结合困难等问题尤为突出，传统“讲授—练习—批改”模式在支撑差异化学习方面存在明显局限[10]。与此同时，大语言模型技术既为程序设计课程带来了重构课堂教学、编程实践与考核评价等环节的契机，也提出了如何引导学生正确、规范使用 AI 工具的新挑战[3]。上述问题在实验课高频答疑、调试指导等场景中表现得尤为集中，教师往往难以及时、逐人回应学生的语法、逻辑与实现细节类问题。

为缓解实验课辅导压力，许多高校在“课堂讲授+编程实验”之外，广泛引入在线评测系统（Online Judge，OJ）或编程训练平台，以自动判题提升练习效率。但现有编程训练系统在智能化、个性化辅导方面仍存在不足，难以在“判对错”之外持续提供过程性、解释性的学习支持[13]。从在线评测系统的研究脉络看，相关工作多聚焦于测试数据评估与生成、判题准确性保障等评测侧问题[24]，对学生在编程学习过程中所需的即时答疑、错误归因与复习引导关注相对有限。因此，在“课堂+OJ”的常用教学组合中，OJ 侧更侧重结果判定，而面向个体差异的过程化辅导仍主要依赖教师人工投入，二者尚难以形成有效互补。

面对平台侧辅导能力不足的现实，部分学生开始直接使用 ChatGPT 等生成式人工智能工具辅助编程学习。相关研究表明，大语言模型能够为学生提供个性化学习路径与编程辅助，在激发学习兴趣、提高学习效率等方面具有明显优势[15]；针对 C 语言程序设计课程的调查亦显示，GenAI 工具在支持个性化学习方面具有积极作用[19]。但与此同时，生成内容的准确性难以保证、学生可能产生过度依赖、代码理解出现偏差、批判性思维受到削弱等问题同样不容忽视[15][19]。换言之，若将通用大模型简单“裸用”于课堂之外，其回答往往难以严格对齐课程大纲、实验要求与教师讲义，更缺乏可核对的知识来源。因此，单纯调用通用大模型，难以满足课程语境下可信、可控的智能辅导要求。

为将大模型能力约束在可信、可溯源的知识范围之内，检索增强生成（Retrieval-Augmented Generation，RAG）成为重要技术路径。研究指出，大语言模型由于训练语料的封闭性与更新滞后，在面对动态事实信息与专业领域知识时，生成结果可能出现偏离事实的情况；RAG 通过集成外部知识库，能够扩展模型知识边界并提升回答质量[18]。在程序设计教学领域，已有学者针对知识幻觉与时效滞后问题，基于课程知识库构建教学辅助智能问答系统，并融合 BM25 与向量检索以提升问答可靠性[1]；在计算机实验教学场景中，亦有研究结合大语言模型与 RAG 技术设计实验指导平台，以缓解预习效率低、实验进度慢等问题，并在实践中取得一定成效[27]。上述探索表明，“课程知识库+RAG+助教平台”的技术路线具有现实可行性；但现有成果多聚焦于单一课程问答、实验指导或训练评测等局部场景，面向多课程、多角色协同的一体化 Web 助教系统仍相对缺乏。

综合上述背景，本课题拟设计并实现智能编程学习助教系统“慧编学伴”。系统面向学生、教师与管理员三类角色，采用 FastAPI、Vue3、MySQL、Redis 与 Chroma 等技术栈，以课程级 RAG 智能对话、结构化代码讲解、错题本与学习仪表盘、教师班级学情视图及 AI 模型与用量治理为主线，在吸收 LLM 与 RAG 领域研究成果的基础上，探索“可信 AI 辅导—学习行为采集—学情反馈”的闭环实现路径；并在课堂教学、编程实践与形成性评价等环节与现有教学模式相衔接[3]，为高校程序设计类课程提供可演示、可扩展的智能助教应用样例。
