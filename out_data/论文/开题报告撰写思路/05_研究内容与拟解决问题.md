二、毕业论文（设计）研究内容、拟解决的主要问题

1、研究内容

本课题以“慧编学伴”智能编程学习助教系统为研究对象，面向高校计算机类程序设计课程，设计并实现覆盖学生、教师、管理员三类角色的 Web 应用。主要研究内容包括：第一，进行需求分析与总体架构设计，采用前后端分离 B/S 架构，后端使用 FastAPI，前端使用 Vue3 与 Element Plus，业务数据存 MySQL，会话与限流使用 Redis，课程资料向量检索使用 Chroma。第二，建设课程知识库与 RAG 对话辅导模块，支持 PDF、TXT、MD、PPTX 等资料上传、解析分块、按课程隔离向量化，并实现带引用来源展示的 SSE 流式多轮问答。第三，实现智能代码讲解模块，支持 C/C++、Java、Python 等代码提交，输出语法、语义、运行三个层次的结构化讲解，并与错题本联动。第四，实现学习分析与推荐模块，包括学习事件采集、错题本、知识点掌握度计算、规则推荐及学生学习仪表盘。第五，实现教师班级学情查看与管理员账户、AI 模型配置、API 密钥加密及 Token 用量统计等功能。第六，开展系统测试与前后端联调，以 pytest 自动化测试和演示脚本验证各模块功能，并继续完善 Docker 等生产部署相关配置。

2、功能

慧编学伴系统采用 B/S 架构，分为学生、教师、管理员三类用户。学生侧重学习与辅导，教师侧重课程与资料维护及班级学情查看，管理员侧重账户与 AI 资源治理。各角色主要功能如下。

学生功能：
（1）登录与个人中心：使用用户名密码登录，获取 JWT 访问令牌；支持刷新令牌与登出；在个人中心修改用户名、密码，查看账户基本信息及本人 AI 调用用量概况。
（2）课程浏览与选课：浏览已发布且审核通过的课程列表，查看课程名称、简介、授课教师等信息；对已发布课程执行选课、退课；在「我的课程」中查看已选课程，作为后续 AI 辅导与学情统计的范围依据。
（3）AI 对话辅导：须先选课方可使用；按所选课程创建、切换与管理多个聊天会话；提问时系统从该课程 Chroma 向量库检索相关片段，经相关性过滤后注入大模型上下文；回答以 SSE 方式流式输出，并在界面展示 Top3 资料引用来源（含片段摘要）；检索无有效命中时提示「当前课程暂无相关资料」，避免脱离课程强行作答；切换课程时可获得与课程相关的推荐问题；对话过程受每用户日调用限流约束。
（4）智能代码讲解：在 Monaco 代码编辑器中选择 C/C++、Python、Java 等语言并提交源码；系统返回语法、语义、运行三个层次的结构化讲解，包括问题列表、修改建议、总评及参考代码（支持一键复制）；运行层分析为模型静态推断，界面标注未真实执行代码；可分页查看本人历史提交记录并回看讲解详情；分析失败或存在较严重语义、运行问题时，结果可自动写入错题本。本功能不绑定具体课程，无需选课即可使用。
（5）错题本：自动归集代码讲解与 AI 对话中的问题记录，按语法错误、语义错误、运行错误、分析失败、对话无上下文等类别分类展示；支持按课程、类型筛选，展开查看具体问题、建议与复习提示；学生可将已理解条目标记为「已掌握」，系统据此更新知识点掌握度。
（6）学习仪表盘：展示近 7 日学习事件趋势、薄弱知识点阶梯图、近期活动摘要；基于规则引擎（非大模型排序）给出薄弱知识点 Top3 及对应 READY 状态复习资料推荐；仪表盘数据采用 Redis 缓存以提升访问性能。

教师功能：
（1）课程管理：创建、编辑、删除（软删除）本人授课课程，维护课程名称、简介等信息；将课程保存为草稿或申请发布；新建与发布申请需管理员审核通过后学生方可选课；管理员可直接创建并发布课程；查看「我的课程」列表及课程状态（草稿/已发布/已归档）。
（2）资料与知识库管理：向本人授课课程上传 PDF、TXT、MD、PPTX 格式教学资料（单文件不超过 10MB）；系统自动完成解析、分块、向量化，资料状态依次经历已上传、解析中、分块中、嵌入中、就绪或失败；失败资料可一键重试；删除资料时同步清理向量片段；在资料列表中查看处理进度、下载原文件；通过格式仓与课程仓管理资料归属，管理员可将资料分派至 Python/Java/C++ 等课程仓。
（3）班级学情：在班级学情页选择本人授课课程，查看选课人数与选课学生名单（含每生本课错题数、未掌握错题数）；查看班级错题类别分布、来源统计及近 7 日新增错题趋势；查看班级薄弱知识点 Top8（基于选课学生掌握度均值）；查看近 7 日全班学习事件趋势及活跃学生数、掌握率等汇总指标；仅可访问本人任课课程，访问他人课程返回权限错误。
（4）个人中心：修改个人信息与密码（与学生、管理员共用能力）。

管理员功能：
（1）管理概览：查看系统内学生数、教师数、课程数等统计信息及近期操作摘要，作为后台运维入口。
（2）学生账户管理：分页查询、新建、编辑、禁用/启用、软删除学生账号，重置密码等。
（3）教师账户管理：分页查询、新建、编辑、禁用/启用、软删除教师账号，重置密码等。
（4）课程与资料：拥有教师全部课程、资料、班级学情能力，且可操作任意课程（不受授课教师限制）；可直接审核或处理教师课程创建、发布申请。
（5）AI 模型管理：查看预置及自定义大语言模型、Embedding 模型列表；切换当前对话模型与向量模型；添加或删除自定义模型；配置 LLM API 密钥（Fernet 加密存储）、Embedding 密钥及每用户日调用上限；保存后立即生效于 AI 对话、代码讲解与向量化任务。
（6）AI 用量监控：查看近 7 日 Token 消耗趋势，按模型、场景（对话/代码讲解等）及学生维度统计调用情况。
（7）操作日志：分页查询登录、账号变更、AI 配置变更等审计记录，便于追溯管理行为。

说明：当前版本教师端不含作业布置、AI 回答审核队列等功能；生产环境 Docker 全栈部署仍在完善中。

3、拟解决的主要问题

（1）课程语境下的可信 AI 辅导问题。通用聊天工具回答易与讲义、实验要求不一致且难以核对依据。本系统通过资料上传、按课程隔离检索、相关性过滤及引用来源展示，使辅导锚定课程内容；无有效检索命中时给出弱提示，避免强行编造。

（2）教学资料到可检索知识库的工程化问题。PDF、PPT 等格式多样，解析与向量化易失败。系统以状态机管理上传、解析、分块、嵌入流程，支持失败重试与软删除同步清理向量，按 course_id 隔离向量集合。

（3）编程实验课过程化辅导不足问题。在线评测侧重判对错，缺少分层说明。系统以结构化 JSON 输出语法、语义、运行三级讲解（运行层为模型静态推断，非沙箱执行），并将严重问题写入错题本。

（4）学情反馈可解释、可验证问题。完全依赖大模型生成学习建议成本高且难测试。系统采集代码讲解与对话等行为，以规则公式计算掌握度，按 Top3 规则推荐复习资料，在学生仪表盘与教师班级视图中可视化展示，便于直观展示数据和系统性查缺补漏。

（5）多角色协同与 AI 调用治理问题。系统采用 JWT 与 RBAC 区分三类用户权限，对 LLM 调用限流与统计，管理员集中配置模型并对 API 密钥加密存储。
