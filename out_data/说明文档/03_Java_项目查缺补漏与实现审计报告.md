| 限流（用户/IP/课程）                    | ⬜   | Redis + 注解或网关             |

### 3.2 核心业务链路（必须打通）

```
[登录] → [选课程] → [上传资料→解析→向量化] → [学生提问→RAG→回答]
                → [学生交代码→（可选运行）→讲解结果落库]
                → [记录事件→更新掌握度→推荐列表]
                → [教师查看统计→审核 AI 回答]
```

| 链路节点 | 关键校验                                   | 状态 |
| -------- | ------------------------------------------ | ---- |
| 登录     | 角色路由、密码加密 BCrypt                  | ⬜   |
| 资料上传 | 文件类型白名单、大小限制、病毒扫描（可选） | ⬜   |
| RAG 问答 | courseId 强制、无资料时降级提示            | ⬜   |
| 代码讲解 | 语言枚举、源码长度上限                     | ⬜   |
| 教师审核 | 未审核内容是否可见的策略实现               | ⬜   |

### 3.3 文档未写但易遗漏的业务规则

1. **学生只能访问已选课程**的数据（垂直越权）。
2. **教师只能管理自己授课课程**的资料与班级。
3. **删除课程**时级联：资料、向量、会话、作业（软删除优先）。
4. **AI 生成内容免责声明**与日志留存（答辩常问）。
5. **并发**：同一学生短时间大量调 LLM 的熔断。

### 5.2 Redis — 建议使用场景

| Key 模式                        | 类型        | TTL         | 用途                 |
| ------------------------------- | ----------- | ----------- | -------------------- |
| `session:token:{userId}`      | String      | 与 JWT 一致 | 单点登录控制（可选） |
| `refresh:token:{uuid}`        | String      | 7d          | 刷新令牌             |
| `rate:llm:{userId}`           | String 计数 | 1d          | LLM 日调用上限       |
| `cache:dashboard:{userId}`    | Hash/JSON   | 10min       | 学生仪表盘           |
| `cache:class_stats:{classId}` | JSON        | 15min       | 教师统计             |
| `ctx:chat:{sessionId}`        | List/JSON   | 2h          | 对话上下文缓存       |
| `lock:material:parse:{id}`    | 分布式锁    | 短          | 防重复解析           |

**Gap**：

- 文档仅写「会话、热点、临时」，**未定义 Key 规范与失效策略**，实现时易混乱。
- 未提及 **缓存与 DB 一致性**（写后删缓存模式）。

### 5.3 向量数据库 — Milvus / FAISS

| ------------- | ---------- | -------------------------------------------------------------------------------------------- |

## 六、跨模块一致性与风险

| 风险                      | 等级 | 说明               | 缓解                                 |
| ------------------------- | ---- | ------------------ | ------------------------------------ |
| LLM 幻觉 / 偏离课程       | 高   | 无 RAG 时体验差    | 强制 courseId + 检索为空则拒答或弱答 |
| API 费用超支              | 高   | 无限额             | Redis 限流 + 管理员配额              |
| 代码执行安全              | 高   | 若本地执行用户代码 | 沙箱隔离，MVP 可不做执行仅静态+LLM   |
| 资料解析失败              | 中   | PPT 复杂           | MVP 支持 PDF/txt/md                  |
| 个人信息与学情隐私        | 中   | 班级排名           | 教师可见明细，学生只见自己           |
| 文档写 MyBatis/JPA 二选一 | 低   | 团队需统一         | 建议 JPA 或 MyBatis-Plus 择一        |
| Python 辅助脚本           | 低   | 部署复杂           | 可后期再加，Java 调 HTTP 即可        |
