## 一、本论题国内外研究动态及研究意义

### （一）研究背景及意义

随着高等教育信息化与人工智能技术的快速发展，计算机类课程已成为高校人才培养体系中的核心组成部分。数据结构、面向对象程序设计、Python 数据分析等课程普遍采用“课堂讲授 + 编程实验 + 在线评测”的教学模式。然而，在实际教学过程中仍面临三方面突出矛盾：一是班级规模扩大导致教师难以及时、逐人解答编程疑问；二是通用搜索引擎与聊天机器人缺乏课程语境，学生获得的答案往往与教学大纲、课堂讲义脱节；三是传统在线评测系统（Online Judge, OJ）侧重“判对错”，对语法错误、逻辑缺陷与调试思路的讲解不足，难以支撑形成性评价与个性化复习。

近年来，以 GPT、通义千问、DeepSeek 等为代表的大语言模型（Large Language Model, LLM）在代码理解、自然语言生成与多轮对话方面展现出较强能力，为构建智能编程助教提供了新的技术路径。与此同时，检索增强生成（Retrieval-Augmented Generation, RAG）通过将外部知识库与生成模型结合，可有效缓解大模型“幻觉”问题，使回答能够引用课程讲义、实验指导书等权威资料。学习分析（Learning Analytics）与教育数据挖掘则为刻画学生薄弱知识点、错题分布与复习路径提供了量化手段。

本课题拟设计并实现“慧编学伴”智能编程学习助教系统，面向高校计算机类课程场景，构建覆盖学生、教师、管理员三类角色的 Web 应用。系统以 FastAPI、Vue3、MySQL、Redis、Chroma 向量数据库及主流 LLM API 为技术基础，实现课程知识库管理、基于 RAG 的 AI 对话辅导、智能代码讲解、学习仪表盘与教师班级学情统计等功能。

**理论意义**：探索 LLM 与 RAG 在高校编程教学中的系统化集成方式，研究课程级知识隔离、引用溯源与相关性过滤等机制对辅导质量的影响；将规则驱动的学习分析与错题本联动相结合，为“AI 辅导—行为采集—学情反馈”闭环提供可复现的工程范式。

**实践意义**：为教师提供可管理的课程资料与班级学情视图，为学生提供 7×24 小时可及的智能辅导与学习档案，为管理员提供 AI 模型与用量管控能力，有助于缓解师资压力、提升实验课辅导效率，并为毕业设计答辩与后续教学平台推广提供可演示、可测试的完整系统。

### （二）国内外研究现状

**1. 智能编程教育与在线辅导系统**

国外较早出现 Codecademy、Khan Academy 等交互式编程学习平台，强调即时反馈与渐进式练习。国内高校与企业在“智慧教学”“在线实验平台”方向持续投入，部分系统已集成自动判题、视频微课与讨论区，但多数仍以结构化题库为中心，对自然语言问答与个性化代码讲解支持有限。

**2. 大语言模型在编程教学中的应用**

研究表明，LLM 可辅助代码解释、错误诊断与示例生成，但也存在答案不可靠、与课程要求不一致等问题。教育领域逐渐从“直接调用通用聊天机器人”转向“领域知识约束 + 可控提示工程 + 引用溯源”的专用助教架构。开源与商业 API（如阿里云百炼、OpenAI 兼容接口）降低了系统落地门槛，但如何在多课程、多角色场景下统一管理模型配置、调用配额与安全策略，仍是工程实践中的难点。

**3. 检索增强生成（RAG）在教育场景的研究**

RAG 通过文档解析、分块、向量化与 Top-K 检索，将生成回答锚定在本地知识库上。教育场景中常见挑战包括：PDF/讲义解析质量、按课程隔离向量空间、检索结果相关性判定，以及多轮对话中上下文与检索结果的协同。本领域研究正从“能否回答问题”转向“能否可追溯地引用教学材料并支持教师审核”。

**4. 学习分析与个性化推荐**

学习分析强调对学习行为数据的采集、建模与可视化。编程类场景中，代码提交、错误类型、AI 对话无上下文等事件可映射为知识点掌握度与错题本条目。相较于依赖 LLM 直接生成学习建议，基于规则引擎与统计聚合的方法具有可解释、可测试、成本低等优势，适合本科毕业设计阶段的可靠实现。

**5. 研究述评**

综合来看，现有研究与产品分别在“智能问答”“自动判题”“学情可视化”等单点能力上较为成熟，但面向高校多课程、多角色、可运维的一体化智能编程助教系统仍相对缺乏。本课题在吸收上述研究成果的基础上，强调 **课程 RAG 辅导 + 结构化代码讲解 + 规则化学情分析 + 教师班级视图 + 管理员 AI 治理** 的端到端闭环，具有明确的工程落地价值。

## 二、毕业论文（设计）研究内容、拟解决的主要问题

### （一）研究内容

本课题以“慧编学伴”系统为研究对象，主要研究内容包括：

1. **需求分析与总体架构设计**  
   面向学生、教师、管理员三类角色，梳理课程管理、资料知识库、AI 辅导、代码讲解、学习分析、账户与 AI 运维等功能需求；设计前后端分离的 B/S 架构及 RESTful API 规范。

2. **课程知识库与 RAG 对话辅导模块**  
   研究 PDF/TXT/MD 等教学资料的解析、分块、向量化与按课程隔离存储；实现基于 Chroma 的语义检索、相关性过滤与 SSE 流式多轮对话，并在回答中展示引用来源。

3. **智能代码讲解模块**  
   研究面向 C/C++、Java、Python 等语言的代码提交与分析流程，设计语法、语义、运行三级结构化讲解方案，并与 Monaco 编辑器集成，形成可理解的调试反馈。

4. **学习分析与推荐模块**  
   研究学习事件埋点、错题本自动归集、知识点掌握度计算与规则推荐算法；实现学生端学习仪表盘（7 日趋势、薄弱知识点、近期活动）及错题本可视化。

5. **教师教学支持与管理模块**  
   实现班级学情概览（选课人数、错题 TOP、薄弱知识点、事件趋势）；实现管理员对学生/教师账户、AI 模型配置、API 密钥加密存储与 Token 用量监控。

6. **系统测试与综合评估**  
   建立以 pytest 为核心的后端自动化测试体系（LLM/Embedding 采用 Mock），开展功能测试、接口测试与答辩演示脚本验证，总结系统优势与不足。

### （二）拟解决的关键问题

1. **课程语境下的可信 AI 辅导问题**  
   如何避免通用大模型回答脱离课程讲义，并使学生能够核对引用来源。

2. **教学资料到可检索知识库的工程化流水线问题**  
   如何实现资料上传、解析、向量化、失败重试与软删除，并保证按课程 ID 隔离向量数据。

3. **超越 OJ 的代码教学反馈问题**  
   如何将 LLM 输出约束为结构化 JSON，形成可分级展示的语法/语义/运行讲解，并与错题本联动。

4. **可解释、可测试的学情建模问题**  
   如何在不依赖 LLM 排序的前提下，基于学习事件与错题数据计算掌握度并生成复习推荐。

5. **多角色权限与 AI 资源治理问题**  
   如何在 JWT + RBAC 体系下隔离学生/教师/管理员能力，并对 AI 调用进行限流、配额与密钥安全管理。

### （三）拟取得的主要创新点

1. **面向课程的知识隔离式 RAG 辅导机制**  
   将向量检索、相关性过滤、引用卡片展示与 SSE 流式输出整合为一体化辅导链路，强化答案与课程资料的可追溯性。

2. **“代码讲解—错题本—掌握度—推荐”规则闭环**  
   以代码分析与对话行为为数据源，自动归集错题本，采用可解释的掌握度公式与 Top3 规则推荐，降低对 LLM 二次调用的依赖。

3. **三级结构化智能代码讲解**  
   将大模型能力约束为语法/语义/运行分层反馈，配合 Monaco 编辑器与参考代码一键复制，提升编程实验课辅导体验。

4. **兼顾教学与管理的多角色平台设计**  
   在同一系统中打通学生学习端、教师班级学情端与管理员 AI/账户治理端，形成可演示、可测试的完整工程样例。

## 三、毕业论文（设计）研究的方法、步骤、措施和研究进度

### （一）研究方法

1. **文献研究法**：检索智能编程教育、LLM 教学应用、RAG、学习分析等相关文献与开源项目，明确技术路线与对比基准。  
2. **需求调研法**：结合高校计算机类课程教学流程与本人实践体验，归纳学生、教师、管理员的核心用例。  
3. **原型迭代法**：采用分阶段（Phase 0～4）增量开发，优先完成学生端 P0 闭环，再扩展教师与管理功能。  
4. **实验测试法**：通过单元测试、接口测试、前后端联调与答辩脚本演示，对功能正确性与稳定性进行验证。

### （二）步骤与技术路线

**总体技术路线**：  
`需求分析 → 架构设计 → 数据库与 API 设计 → 后端核心服务实现 → 前端页面开发 → 联调测试 → 部署优化 → 论文撰写`

**系统技术架构**：

- **前端**：Vue 3 + Element Plus + ECharts + Monaco Editor  
- **后端**：FastAPI + SQLAlchemy + Alembic + JWT 认证  
- **数据层**：MySQL（业务数据）、Redis（会话黑名单/限流/缓存）、Chroma（向量检索）  
- **AI 层**：可配置 LLM API（如百炼/DeepSeek 兼容接口）+ Embedding 服务 + RAG 流水线  
- **运维**：Docker Compose 中间件；生产部署待完善

```
        ↓

        ↓
pytest 自动化测试（154 项）+ 答辩演示脚本验证
        ↓
部署优化（M08）+ 论文撰写与答辩准备
```

**关键措施**：

- 模块开发前编写 `docs/modules/Mxx_*.md` 说明文档，明确验收标准；  
- 外部 LLM/Embedding 在测试中统一 Mock，保证 CI 可重复；  
- 提供 `seed_demo.py`、`seed_dashboard_demo.py` 等脚本，保障 15 分钟答辩可复现；  
- 统一 API 响应格式、错误码与端口配置，降低联调成本。
