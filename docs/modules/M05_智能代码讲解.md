# M05_智能代码讲解

> **Phase**：3 | **优先级**：P0 | **更新**：2026-06-10（文档与实现同步）

## 1. 业务目标

学生提交编程练习代码后，系统提供 **语法 / 语义 / 运行** 三级结构化讲解，超越 OJ「只判对错」，给出可理解的调试与教学反馈。**不依赖课程资料与选课**，作为独立练习入口。

## 2. 用例与角色

| 用例 | 角色 | 说明 |
|------|------|------|
| 提交代码分析 | student | 同步分析并落库 |
| 查看分析结果 | student / admin | 按 `submission_id` |
| 提交历史列表 | student | 按当前用户分页 |
| 查看历史详情 | student | 历史页跳转主页面并加载记录 |

## 3. 业务规则

1. **不绑定课程**：`course_id` 固定为 `NULL`；无需选课即可使用
2. **语言**：支持 `c`、`cpp`、`python`、`java`（前端提交前启发式校验语言与代码一致性）
3. 源码长度上限 `code_max_source_chars`（默认 16384），超出截断分析并在结果中标记 `truncated`
4. 禁止 `eval/exec` 等危险输入；命中则分析 `failed`
5. 每次分析计入 LLM 日限流（Redis `rate:llm:{userId}`）
6. **运行级**为静态推断，界面标注「未真实执行」；答辩如实说明
7. `version` 为同一用户全局提交序号（非按课计数）

## 4. 数据库表

| 表 | 核心字段 | 逻辑外键 |
|----|---------|---------|
| `code_submission` | user_id, course_id(NULL), assignment_id, language, source_code, version | user.id |
| `analysis_result` | submission_id, status, result_json, error_message | code_submission.id（1:1） |

| 迁移 | 说明 |
|------|------|
| `phase5_code_analysis.py` | 初版建表 |
| `phase6_code_optional_course.py` | `course_id` 可空；语言枚举增加 `c` |

启动时 `ensure_code_schema()` 可在未跑 Alembic 时补建表（MySQL）。

## 5. API 列表

| 方法 | 路径 | 角色 | 说明 |
|------|------|------|------|
| POST | `/api/v1/code/submit` | student | Body: `language`, `source_code` |
| GET | `/api/v1/code/submit/{id}/result` | student / admin | 提交详情 + 分析结果 |
| GET | `/api/v1/code/submissions` | student | `page_num`, `page_size`（无 `course_id`） |

### 5.1 提交请求示例

```json
{
  "language": "python",
  "source_code": "print(1)"
}
```

### 5.2 结果 JSON（`result_json`）

见 `app/schemas/code_analysis.py`：`AnalysisResultData`

- `summary`：总评（多句）
- `levels.syntax / semantic / runtime`：`score`、`issues[]`、`suggestions[]`
- `fixed_code`：参考代码（前端「一键复制」）
- `examples`：常见错误参考

## 6. 前端

| 路由 | 说明 |
|------|------|
| `/student/code` | 语言选择、可横向滚动编辑器、可拖拽分栏、讲解结果滚动区 |
| `/student/code/history` | 提交历史表格；点击「查看」回到主页面并带 `?id=` |

侧栏菜单：**代码讲解**。

### 6.1 交互要点

- 提交前：代码与所选语言不一致 → 弹窗提示并可选切换语言
- 讲解结果：总评 / 问题卡片分行展示；参考代码支持一键复制
- 编辑器与结果区：`SplitPane` 拖拽调整宽度占比

## 7. 测试用例

## 8. 与 M06 衔接（预留）
