# M02_课程与选课

> **Phase**：1 | **优先级**：P0 | **更新**：2026-06-08

## 1. 业务目标

教师/管理员创建与管理课程；学生浏览已发布课程并选课；各角色查看「我的课程」列表。

## 2. 用例与角色

| 用例 | 角色 | 说明 |
|------|------|------|
| 创建课程 | teacher, admin | 教师默认为自己授课 |
| 编辑/删除课程 | 授课教师, admin | 软删除 |
| 课程列表（管理） | teacher, admin | 分页 |
| 学生选课 | student | 对已发布课程 join |
| 我的课程 | 全部登录用户 | 按角色返回不同列表 |

## 3. 业务规则与状态机

### 3.1 课程状态

```
draft → published → archived
```

| status | 说明 |
|--------|------|
| `draft` | 草稿，学生不可见、不可选 |
| `published` | 已发布，学生可选课 |
| `archived` | 已归档，不可新选 |

### 3.2 业务规则

1. 学生只能 join `published` 且 `create_approval=approved` 的未删除课程
2. 重复选课返回业务错误（已选过）
3. 教师只能 CRUD 自己 `teacher_id` 的课程；admin 可操作全部
4. 删除课程为软删除（`deleted=1`）
5. `course_teacher` 表预留协同授课，MVP 创建课程时同步写入主讲教师
6. **授课教师姓名**通过 `teacher_id` 逻辑外键关联 `user.username` 查询返回
7. **教师新建/发布审核**：教师创建课程或申请发布需管理员审核；管理员可直接创建/发布
8. **课程发布时间** `published_at` 在审核通过发布或管理员直接发布时写入，格式 `yyyy/mm/dd HH-MM-SS`

### 3.3 审核状态

| 字段 | 取值 | 说明 |
|------|------|------|
| `create_approval` | pending / approved / rejected | 新建审核（教师提交后 pending） |
| `publish_approval` | none / pending / approved / rejected | 发布审核（教师申请发布后 pending） |

## 4. 数据库表（逻辑外键）

| 表 | 核心字段 | 逻辑外键 |
|----|---------|---------|
| `course` | name, description, teacher_id, status, create_approval, publish_approval, published_at | user.id → teacher_id |
| `course_student` | course_id, user_id, joined_at | course.id, user.id |
| `course_teacher` | course_id, user_id | course.id, user.id |

## 5. API 列表

| 方法 | 路径 | 角色 | 说明 |
|------|------|------|------|
| GET | `/api/v1/courses` | teacher, admin | 分页列表 |
| POST | `/api/v1/courses` | teacher, admin | 创建 |
| GET | `/api/v1/courses/{id}` | 登录 | 详情 |
| PUT | `/api/v1/courses/{id}` | 授课教师, admin | 更新 |
| DELETE | `/api/v1/courses/{id}` | 授课教师, admin | 软删除 |
| POST | `/api/v1/courses/{id}/request-publish` | teacher | 申请发布 |
| POST | `/api/v1/courses/{id}/approve-create` | admin | 新建审核 |
| POST | `/api/v1/courses/{id}/approve-publish` | admin | 发布审核 |
| POST | `/api/v1/courses/{id}/join` | student | 选课 |
| POST | `/api/v1/courses/{id}/leave` | student | 退课 |
| GET | `/api/v1/courses/my` | 登录 | 我的课程 |

## 7. 安全、限流与异常

- 学生访问 POST/PUT/DELETE `/courses` → 40301
- 访问不存在或已删除课程 → 40401
- 垂直越权：学生不能修改他人选课记录
