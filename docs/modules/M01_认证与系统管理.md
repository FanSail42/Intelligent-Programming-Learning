# M01_认证与系统管理

> **Phase**：1 | **优先级**：P0 | **更新**：2026-06-08

## 1. 业务目标

提供用户身份认证、JWT 令牌管理、角色鉴权（RBAC）及系统配置/操作日志基础能力，支撑学生/教师/管理员三端访问控制。

## 2. 用例与角色

| 用例 | 角色 | 说明 |
|------|------|------|
| 登录 | 公开 | 用户名+密码，返回 access/refresh token |
| 刷新令牌 | 公开 | refresh_token 换取新 access_token |
| 登出 | 登录用户 | access_token jti 入 Redis 黑名单 |
| 获取当前用户 | 登录用户 | 返回 id/username/role/status |
| **个人中心** | 登录用户 | 改用户名/密码、账户概览、学生 AI 用量 |
| 系统配置读写 | 管理员 | M09 AI 模型管理（sys_config + Fernet） |
| 用户账号 CRUD / 操作日志 | 管理员 | 见 **M09 账户管理**（Phase 4 已交付） |

## 3. 业务规则与状态机

### 3.1 用户角色

| role | 说明 |
|------|------|
| `student` | 学生，可选课、学习 |
| `teacher` | 教师，可管理自己的课程 |
| `admin` | 管理员，可管理全部课程与用户 |

### 3.2 用户状态

| status | 说明 |
|--------|------|
| `active` | 可登录 |
| `disabled` | 禁止登录 |

### 3.3 Token 规则

- access_token 有效期：30 分钟（可配置）
- refresh_token 有效期：7 天，存 Redis `refresh:token:{jti}`
- 登出：access jti 写入 `jwt:blacklist:{jti}`，TTL = 剩余有效期
- 密码：bcrypt 哈希，日志禁止输出明文

## 7. 安全、限流与异常

- 登录失败统一提示「用户名或密码错误」，不暴露用户是否存在
- **access_token 有效期 3 天**（`JWT_ACCESS_TOKEN_EXPIRE_MINUTES=4320`），减少演示/学习场景频繁重新登录
- refresh_token 有效期 7 天，一次性轮换（刷新后旧 jti 失效）
- JWT 校验失败返回 `code: 40101`
- 角色不足返回 `code: 40301`

## 8. 测试用例

| 用例 | 预期 |
