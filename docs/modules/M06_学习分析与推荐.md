# M06_学习分析与推荐

> **Phase**：3 | **优先级**：P1 | **更新**：2026-06-10（**已编码验收**）

## 1. 业务目标

采集学生在 **代码讲解、AI 对话、资料浏览** 等行为，形成 **错题本** 与 **知识点掌握度**，基于 **规则引擎**（非 LLM 排序）给出复习建议与资料推荐，支撑学生端学习仪表盘。

## 2. 用例与角色

| 用例 | 角色 | 说明 |
|------|------|------|
| 行为埋点 | student / 系统内部 | 记录学习事件 |
| 查看学习仪表盘 | student | 趋势、薄弱知识点、简要统计 |
| 错题本列表 | student | 按课程/类型/类别筛选，展开查看详细分析 |
| 错题本统计 | student | 类别分布、来源、趋势等 ECharts 可视化 |
| 标记错题已掌握 | student | 更新掌握状态并刷新掌握度 |
| 获取复习推荐 | student | 薄弱 KP + 推荐资料 ID 列表 |

教师端学情统计归属 **M07（Phase 4）**；本模块聚焦 **学生侧** 档案与推荐。

## 3. 业务规则

### 3.1 事件采集

| event_type | 触发时机 | 说明 |
|------------|---------|------|
| `code_submit` | M05 提交成功 | payload: submission_id, language |
| `code_analysis_error` | M05 分析 failed 或 semantic/runtime 为 error | 自动进错题本 |
| `chat_message` | M04 学生发问 | payload: session_id, message_id |
| `chat_no_context` | M04 回答 `no_context=true` | 可选进错题本（弱相关） |
| `material_view` | 学生打开资料 | Phase 3 可选，P1 可延后 |

- `course_id`：与业务上下文一致；M05 无课程时可为 `NULL`
- `kp_id`：能映射则写，否则 `NULL`（MVP 可先不建完整 KP 树，用占位 KP）

### 3.2 错题本

| source_type | ref_id 含义 |
|-------------|------------|
| `code_submission` | `code_submission.id` |
| `chat_message` | `chat_message.id` |

- 同一 `user + source_type + ref_id` 不重复插入（幂等）
- `mastered=0` 默认；学生可标记 `mastered=1`

**错题类别**（`app/services/wrong_book_analysis.py`，基于 M05 分析结果或对话上下文自动推断）：

| category | 说明 |
|----------|------|
| `syntax_error` | 语法错误 |
| `semantic_error` | 语义/逻辑错误 |
| `runtime_error` | 运行期错误 |
| `analysis_failed` | 分析失败 |
| `chat_no_context` | AI 对话无资料上下文 |
| `unknown` | 无法归类 |

列表项扩展字段：`category`、`category_label`、`issues`、`suggestions`、`review_tip`、`kp_name`、`has_fixed_code`。

### 3.3 掌握度（规则，不用 LLM）

服务：`app/services/mastery.py`

**知识点得分** `score`（0～100，越高越好）建议公式：

```
对每个关联错题（未掌握）：
  weight = 1.0（code_error）或 0.6（chat_no_context）
  days = (now - created_at).days
  penalty = weight * exp(-days / 30)   # 时间越久略衰减关注权重

kp_score = max(0, 100 - sum(penalty) * 20)
```

- 标记 `mastered` 后该错题不再计入 penalty
- 单元测试固定输入/输出，禁止依赖随机或 LLM

### 3.4 推荐（规则）

`GET /recommendations?course_id=`：

1. 取该课（或全局）掌握度最低的 Top3 `kp_id`
2. 每 KP 推荐 1 条 **已 READY** 的 `course_material.id`（同 course，按上传时间）
3. 无 KP 时：推荐该课最近 READY 资料
4. 仅返回学生 **已选课** 或 **全校公开浏览课** 范围内的数据

**禁止** 用 LLM 做推荐排序；可选 LLM 仅润色推荐文案（Phase 4+）。

### 3.5 缓存

| Key | TTL | 失效 |
|-----|-----|------|
| `cache:dashboard:{userId}` | 10min | 写 `learning_event` / 更新错题本后 DEL |

## 4. 数据库表

| 表 | 核心字段 | 逻辑外键 |
|----|---------|---------|
| `knowledge_point` | course_id, parent_id, name, sort_order | course.id；parent 自引用 |
| `learning_event` | user_id, course_id, event_type, kp_id, payload_json | user.id, course.id, knowledge_point.id |
| `wrong_question_book` | user_id, course_id, source_type, ref_id, kp_id, mastered | user.id；ref 依 source_type |
| `user_kp_mastery` | user_id, kp_id, score, updated_at | user.id, knowledge_point.id |

公共字段：`id, created_at, updated_at, deleted`（与项目统一）

**索引建议**：

- `learning_event(user_id, created_at)`
- `wrong_question_book(user_id, mastered, course_id)`
- `user_kp_mastery(user_id, kp_id)` UNIQUE

迁移：新建 `alembic/versions/phase7_learning_analysis.py`（命名以实际为准）

## 5. API 列表

| 方法 | 路径 | 角色 | 说明 |
|------|------|------|------|
| POST | `/api/v1/learning/events` | student / 内部 | 埋点（学生手动埋点少用，主要由 M04/M05 服务调用） |
| GET | `/api/v1/learning/dashboard` | student | 仪表盘聚合数据 |
| GET | `/api/v1/learning/wrong-book/stats` | student | 统计聚合；`course_id?`, `days?`（默认 30） |
| GET | `/api/v1/learning/wrong-book` | student | 分页；`course_id?`, `mastered?`, `category?`, `page_num`, `page_size` |
| PUT | `/api/v1/learning/wrong-book/{id}/mastered` | student | Body: `mastered: bool` |
| GET | `/api/v1/learning/recommendations` | student | `course_id` 必填 |
