# M03_课程知识库

> **Phase**：2 | **优先级**：P0 | **更新**：2026-06-09

## 1. 业务目标

教师上传 PDF/TXT/MD 课程资料，系统自动解析、切片、向量化，供 RAG 问答检索。

## 2. 用例与角色

| 用例 | 角色 | 说明 |
|------|------|------|
| 上传资料 | teacher, admin | multipart 上传 |
| 查看资料列表 | teacher, admin | 按 course_id |
| 轮询处理状态 | teacher, admin | UPLOADED→…→READY |

## 3. 业务规则与状态机

```
UPLOADED → PARSING → CHUNKING → EMBEDDING → READY
                                    ↘ FAILED（可重试）
```

- 白名单：pdf、txt、md；单文件 ≤ 20MB
- 教师只能向自己授课课程上传；admin 不限
- 删除为软删除，向量库同步删除 chunk

## 4. 数据库表（逻辑外键）

| 表 | 核心字段 | 逻辑外键 |
|----|---------|---------|
| `course_material` | course_id, warehouse_id, uploaded_by, type, file_path, status, version | course.id, material_warehouse.id, user.id |
| `material_warehouse` | warehouse_kind, course_subject, material_type, name | — |
| `material_chunk` | material_id, seq, text, source_page, token_count | course_material.id |

ChromaDB metadata：`course_id`, `chunk_id`, `page`

## 5. API 列表

| 方法 | 路径 | 角色 |
|------|------|------|
| POST | `/api/v1/materials/upload` | teacher, admin |
| GET | `/api/v1/materials/{id}/status` | teacher, admin |
| GET | `/api/v1/materials` | teacher, admin |
| POST | `/api/v1/materials/{id}/retry` | teacher, admin（仅 FAILED） |
| DELETE | `/api/v1/materials/{id}` | teacher, admin（软删除 + 删向量） |
| GET | `/api/v1/materials/{id}/download` | teacher, admin（下载原文件到本地，**不对学生开放**） |

## 6. 前端

| 路由 | 说明 |
|------|------|
| `/teacher/materials` | 上传 + 状态 Tag + 删除二次确认 |
| `/teacher/warehouses` | 格式仓/课程仓分区展示 |
| `/teacher/warehouses/:id` | 仓内资料分页搜索、**下载**；课程仓管理员分派 |

上传时按文件格式自动归入格式仓库；Python/Java/C++ 课程仓由管理员手动分派。

## 7. 安全与边界

- 文件类型与大小校验
- 路径 traversal 防护（仅存 UUID 文件名）
- 异步任务失败写 FAILED + Redis 缓存状态

## 8. 测试用例

- 上传 PDF → Mock 解析 → 切片数量 > 0
- 非授课教师上传 → 403
- vector_store 按 course_id 隔离
